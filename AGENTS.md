# Git Workflow Standards
You MUST follow professional DevOps standards for all code changes:
1. Always create a new branch for changes (e.g., `feat/xxx`, `fix/xxx`, `chore/xxx`).
2. Make your code changes and verify them.
3. Commit using Conventional Commits (e.g., `feat: added feature`, `fix: resolved issue`).
4. Push the branch to the remote repository.
5. Notify the user to create a Pull Request (PR) to merge into the main branch.
